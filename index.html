DDL, or Data Definition Language, is a set of SQL commands used to create, modify, and delete database objects such as tables, views, indexes, and sequences.

Here are some of the most common DDL commands and their explanations:

CREATE: This command is used to create new objects in a database. The CREATE command can be used to create tables, views, indexes, and other objects.

Example:

  CREATE TABLE customers (
  id INT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  email VARCHAR(50) UNIQUE
  );


ALTER: This command is used to modify the structure of existing database objects. The ALTER command can be used to add or remove columns, modify data types, or change constraints.

Example:
  ALTER TABLE customers
  ADD COLUMN phone VARCHAR(15);


DROP: This command is used to delete database objects. The DROP command can be used to delete tables, views, indexes, and other objects.

Example:
  DROP TABLE customers;


RENAME: This command is used to change the name of a database object. The RENAME command can be used to rename tables and other objects.

Example:
  ALTER TABLE customers
  RENAME TO clients;


TRUNCATE: This command is used to delete all the data in a table, while keeping the table structure intact.

Example:
  TRUNCATE TABLE clients;

DML stands for Data Manipulation Language and it is a subset of SQL commands that are used to manipulate data in a database. DML commands are used to insert, update, and delete data from tables. Here are some of the most common DML commands and their explanations:

  • INSERT: This command is used to insert new data into a table. The INSERT command can be used to insert one or more rows of data into a table.

    Example:
    INSERT INTO customers (name, email, phone)
    VALUES ('John Doe', 'john.doe@example.com', '123-456-7890');
  • UPDATE: This command is used to modify existing data in a table. The UPDATE command can be used to update one or more rows of data in a table.

    Example:
    UPDATE customers
    SET phone = '555-555-5555'
    WHERE name = 'John Doe';
  • DELETE: This command is used to delete data from a table. The DELETE command can be used to delete one or more rows of data from a table.

    Example:
    DELETE FROM customers
    WHERE email = 'john.doe@example.com';

These are just a few examples of DML commands. There are many more commands available, and they can vary slightly depending on the specific database management system being used.

TCL, or Transaction Control Language, is a subset of SQL commands that are used to manage transactions in a database. TCL commands allow you to control the transactional behavior of SQL statements. Here are some of the most common TCL commands and their explanations:

COMMIT: This command is used to permanently save the changes made in a transaction to the database. Once the COMMIT command is executed, the changes become permanent and cannot be rolled back.

Example:

BEGIN TRANSACTION;
-- Some SQL statements
COMMIT;

ROLLBACK: This command is used to undo the changes made in a transaction and restore the database to its previous state. The ROLLBACK command can be used to undo the changes made by one or more SQL statements in a transaction.

Example:

BEGIN TRANSACTION;
ROLLBACK;

SAVEPOINT: This command is used to create a savepoint within a transaction. A savepoint is a point in a transaction where you can roll back to without rolling back the entire transaction.

Example:
BEGIN TRANSACTION;
SAVEPOINT my_savepoint;
ROLLBACK TO my_savepoint;
COMMIT;

These are just a few examples of TCL commands. There are many more commands available, and they can vary slightly depending on the specific database management system being used.

  1. SQL> COMMIT;
    Commit complete.
  2. SQL> ROLLBACK;
    Rollback complete.
  3. SQL> SAVEPOINT my_savepoint;
    Savepoint created.



In SQL, a join is used to combine rows from two or more tables based on a related column between them. Here are some of the most common types of joins and their explanations:

Inner join:

This type of join is used to combine rows from two tables based on a related column. An inner join returns only the rows that have matching values in both tables.


      Example:
      -- create table1 with random values
      CREATE TABLE table1 (
        id NUMBER(10),
        value1 VARCHAR2(10),
        value2 NUMBER(10,2)
      );
      -- create table2 with random values
      CREATE TABLE table2 (
        id NUMBER(10),
        value3 VARCHAR2(10),
        value4 NUMBER(10,2)
      );
      -- create table3 with random values
      CREATE TABLE table3 (
        id NUMBER(10),
        value5 VARCHAR2(11),
        value6 NUMBER(10,2)
      );
      -- perform inner join between table1, table2, and table3
      SELECT *
      FROM table1
      JOIN table2 ON table1.id = table2.id
      JOIN table3 ON table1.id = table3.id;

Outer-join:

This type of join is used to combine rows from two tables based on a related column. An outer join returns all the rows from one table and matching rows from the other table, or NULL values if there is no match.


      Example:
      -- Create the first table
      CREATE TABLE orders (
        order_id NUMBER,
        customer_id NUMBER,
        order_date DATE,
        total_amount NUMBER
      );
      -- Create the second table
      CREATE TABLE customers (
        customer_id NUMBER,
        first_name VARCHAR2(11),
        last_name VARCHAR2(11),
        email VARCHAR2(11)
      );
      -- Perform a left outer join on the orders and customers tables
      SELECT o.order_id, c.first_name, c.last_name, o.total_amount
      FROM orders o
      LEFT OUTER JOIN customers c
      ON o.customer_id = c.customer_id;

Cartesian-join:

This type of join is also called a cross join. A Cartesian join returns all possible combinations of rows from two tables, without any condition.


      Example:
      -- Create the first table
      CREATE TABLE cities (
        city_name VARCHAR2(11),
        country_name VARCHAR2(11)
      );
      -- Create the second table
      CREATE TABLE colors (
        color_name VARCHAR2(11)
      );
      -- Perform a Cartesian join on the cities and colors tables
      SELECT c.city_name, c.country_name, o.color_name
      FROM cities c
      CROSS JOIN colors o;

Equi-join:

This type of join is used to combine rows from two tables based on a matching column. In an equi-join, the matching column has the same name and data type in both tables.


      Example:
      -- Create the first table
      CREATE TABLE employees (
        employee_id NUMBER,
        first_name VARCHAR2(11),
        last_name VARCHAR2(11),
        department_id NUMBER

SELECT name, grade
FROM students
WHERE grade >= ALL (SELECT grade FROM students WHERE age > 18);
Output:
+-------+-------+
| name | grade |
+-------+-------+
| Alice | 3.5 |
| Eve | 3.8 |
+-------+-------+
In this example, we selected the names and grades of all students whose grade is greater than or equal to all grades of students who are older than 18 years old.

Now, let's perform the UNION, INTERSECT, and MINUS operations:
UNION operation:
The UNION operator is used to combine the result sets of two or more SELECT statements into a single result set.
Example:
sql Copy code SELECT name FROM fruits
UNION
SELECT name FROM vegetables;
Output:
diff Copy code +----------+
| name |
+----------+
| Apple |
| Banana |
| Orange |
| Grape |
| Mango |
| Carrot |
| Broccoli |
| Tomato |
| Lettuce |
| Cucumber |
+----------+
In this example, we selected the names of all fruits and vegetables using the UNION operator.
INTERSECT operation:
The INTERSECT operator is used to select only the common rows returned by two SELECT statements.
Example:
sql Copy code SELECT name FROM fruits
INTERSECT
SELECT name FROM vegetables;
Output:
sql Copy code
+-------+
| name |
+-------+
| None |
+-------+
In this example, we selected the names of all fruits and vegetables using the INTERSECT operator. However, since there are no common fruits and vegetables, the result set is empty.
MINUS operation:
The MINUS operator is used to select all rows returned by the first SELECT statement but not by the second SELECT statement.
Example:
sql Copy code SELECT name FROM fruits
MINUS
SELECT name FROM vegetables;
Output:
diff Copy code
+--------+
| name |
+--------+

| Banana |
| Orange |

+--------+
In this example, we selected the names of all fruits that are not also vegetables using the MINUS operator.
